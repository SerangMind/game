## 현재 게임 컨셉

- 바이브 코딩 용어 설명을 보고, 위에서 떨어지는 용어 떡 중 정답을 받아먹는 게임.
- 글자 수집 모드는 제거했고, 현재는 문항형만 사용.
- 문제는 게임 시작 전에는 표시하지 않고 `바이브 코딩 용어 게임` 제목만 보여준다.
- 시작 후에는 문제 문장을 가운데 크게 표시한다.
- 문제 순서는 게임 시작마다 랜덤이며, 한 판 안에서는 중복 없이 6개 용어가 나온다.

## 최근 반영된 규칙

- 정답 떡은 라운드 초반에 바로 나오지 않는다.
- 각 라운드 초반에는 오답 떡이 먼저 나온다.
- 정답 떡은 화면에 동시에 1개만 떨어진다.
- 오답 떡을 받아먹으면 생명이 줄어든다.
- 정답 떡을 놓쳐도 생명은 줄어들지 않는다.
- 정답 떡을 받으면 점수가 오르고 다음 라운드로 넘어간다.

## 현재 UI 상태

- 상단 제목 문구는 제거했다.
- 상단에는 `점수`, `생명`, `라운드`만 크게 표시한다.
- 문제 영역에는 `문항 떡` 배지를 제거했다.
- 문제 문장은 가운데 정렬이며 크게 표시한다.
- 떨어지는 용어 떡은 말랑한 흰 찹쌀떡 타원형으로 표시한다.
- 캐릭터는 `assets/병아리.png`를 사용한다.
- 말풍선과 `삐약!` 반응은 제거했다.

## 주요 파일

- `index.html`: 화면 구조와 캐릭터 이미지 경로.
- `src/main.js`: 게임 데이터, 랜덤 문제 순서, 떡 생성, 충돌, 점수/생명 로직.
- `src/styles.css`: HUD, 문제 영역, 떡 모양, 캐릭터 크기와 전체 레이아웃.
- `assets/병아리.png`: 현재 병아리 캐릭터 PNG.
- `tests/playwright-vibe-check.mjs`: 자동 플레이 테스트.
- `tools/serve.mjs`: 로컬 정적 서버.
